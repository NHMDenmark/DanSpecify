In some cases an attachment can be related to multiple collection objects - for example a publication that describes multiple specimens.
In these cases it's possible to upload a file which will be referenced by all related objects.

### 1. Create a Reference work

<img src="https://github.com/user-attachments/assets/87af024e-ba44-448f-830c-f62840f33e8c"
     alt="Reference_Work"
     width="400" />

Choose the type and the title of the image/attachment,
then add your attachment by clicking on the small RW icon.

<img src="https://github.com/user-attachments/assets/47c70d06-57c4-4037-aca6-a69050278b69"
     alt="Reference_Work_second_step"
     width="600" />

Save your Reference work, and remember the Title of your Refence work - you will need it in the next step.

### 2. Create a query to find the objects sharing the same attachment

Create a query to find the objects to which you want to attach the same attachment, be as specific as needed.
Add one more filter to your query: the Title of the Reference Work, but leave it as 'Any'.

<img src="https://github.com/user-attachments/assets/fb9321e6-cad7-4683-8502-f36782db5297"
     alt="Query"
     width="600" />

From here you can continue two different ways:
Batch Edit works nicely, if you have no existing Citation/Reference Work attached to your Collection Object yet. --> Continue with Point 3.
If you have existing Reference Work attached to your Collection Object(s), then you need to use WorkBench. --> Continue with Point 4.


### 3. Batch Edit

Save your query, run it, then click on Batch Edit above the Results of the query.
Let it add further ranks, if it is necesserary.

<img src="https://github.com/user-attachments/assets/39128b67-145c-414e-a043-8703af27f776"
     alt="Additional ranks"
     width="400" />

Add the Title of your Reference work to all your objects. 

<img src="https://github.com/user-attachments/assets/b0952072-00ed-41bf-af64-e295d2d06236"
     alt="BE"
     width="600" />

Save, Validate and if there is no error founded - Commit it. 

### 4. WorkBench

Run your query, export your data as a csv, including the catalogue number of your objects.
In your preferred spreadsheet application (Excel, Google Sheets,...) create a spreadsheet with the following columns:
Catalogue Number, Title of your Reference work, Type of your Reference work.

<img src="https://github.com/user-attachments/assets/c00bb701-8e3d-4586-bb9f-70c56a57b180"
     alt="spreadsheet"
     width="600" />

Back to Specify, click on WorkBench, then Import File.
Find your pre-made spreadsheet and import it.

Tou need to choosed Collection Object Citation as a base table, when you're creating the Upload Plan.

<img src="https://github.com/user-attachments/assets/4b202873-34e2-40ab-a884-2c63a1549ea2"
     alt="Base table"
     width="400" />

Your upload plan should look like this:


<img src="https://github.com/user-attachments/assets/ccd8a915-e655-4901-86f8-a2931d5bcf40"
     alt="spreadsheet"
     width="600" />

Then you need to save, validate and finally upload your dataset.

### 5. Check your work

You should find your shared attachment linked to every collection object you batch edited/imported as a WorkBench dataset in the previous step.
(The location of your Reference work can be varied based on your collection - ask your colleagues or contact your support team, if you cannot find it on your form.)
